# 4️⃣ 与插件联动的foobar配置 😎

## 1、无歌词设置（纯音乐）

没有歌词的时候布局会居中并隐藏歌词组件

需要满足三个条件

1. `%UNSYNCED LYRICS%` `%LYRICS%` 两个歌词标签中，其中一个里面有`纯音乐`这三个字

![image](https://github.com/user-attachments/assets/eb1c14f0-e9ae-4c61-a83e-0e1195588236)

2.不能显示歌词

当歌词里面有`纯音乐`这三个字的时候，可以去歌词处理添加`*纯音乐*` 

![image](https://github.com/user-attachments/assets/a34173bb-c365-46bb-86b7-95f738b1cf8d)

3.不能搜索歌词

需要在搜索过滤添加关键词  `%UNSYNCED LYRICS%` `%LYRICS%`  通配符匹配  `*纯音乐*`

当这两个本地标签的歌词里面有（纯音乐）这三个字时，会停止搜索歌词

![image](https://github.com/user-attachments/assets/c879d358-8098-48d5-81a1-6cf207041fe9)

## 3. 【实验性功能】歌词关闭按钮（仅限横屏布局）  
- 在横屏布局中新增了歌词关闭按钮。  
- 要启用该功能，需要在 **歌词处理规则** 中添加一条规则：选择 **通配符移除**，并添加 `*`，否则该功能不会生效。  
- 该功能 **默认关闭**`visibility="0"`，目前仍然 **不够成熟**。  
- 如需使用，需要手动在对应的 `.xml` 布局文件中开启`visibility="1"`。  

已知问题说明 :
该功能是通过规则开关过滤所有歌词来实现的，因此会导致 `%esl_has_playing_lyric%` 字段无法检测到歌词（所有字符都被过滤掉了）。而歌词开关按钮正是通过该字段来判断是否显示。  

在规则开启的状态下切歌，可能会导致按钮不再出现。此时需要在歌词面板 **右键 → 歌词规则菜单** 中关闭 `*` 这条规则，按钮才会重新显示。  

根据是否有歌词显示按钮状态的条件为：  
```xml
visibility="$ifequal(%esl_has_playing_lyric%,1,1,0)"
```

还有一种做法是始终显示按钮：  
```xml
visibility="1"
```
但这种方式在某些情况下无法正确判断功能的开关状态。
